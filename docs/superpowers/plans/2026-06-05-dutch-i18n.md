# Dutch i18n (Android app) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the Android app available in Dutch as well as English, following the system locale (no in-app picker).

**Architecture:** Extract every hardcoded user-facing string into `res/values/strings.xml` (English base) and translate them in `res/values-nl/strings.xml`; replace the Compose literals with `stringResource(...)`. The system locale selects the set automatically. Server-provided text (device names, server error messages) stays as-is.

**Tech Stack:** Android resources, Jetpack Compose `androidx.compose.ui.res.stringResource`, `androidx.annotation.StringRes`. Build (from `android/`): `./gradlew :app:testDebugUnitTest :app:compileDebugKotlin` / `:app:assembleDebug`.

**Spec:** `docs/superpowers/specs/2026-06-04-app-i18n-dutch-design.md`

**Convention:** `R` = `io.konektis.ems.R`; add `import io.konektis.ems.R` and `import androidx.compose.ui.res.stringResource` to each Compose file that gains a `stringResource(...)` call. Format strings use positional args (`%1$s`, `%1$d`). The `●` dot, the `—` em-dash placeholder, and the `EMS` brand name stay as literals (not translated).

---

## File Structure

- `android/app/src/main/res/values/strings.xml` — English base (all keys).
- `android/app/src/main/res/values-nl/strings.xml` — Dutch overlay (same keys).
- Compose screens/components that lose their literals: `ui/settings/SettingsScreen.kt`,
  `ui/dashboard/DashboardScreen.kt`, `ui/charger/ChargerScreen.kt`, `ui/charger/ChargerScene.kt`,
  `ui/heatpump/HeatPumpScreen.kt`, `ui/overview/OverviewScreen.kt`,
  `ui/components/DeviceCard.kt`, `ui/components/EnergyFlowDiagram.kt`.

`StatusHero.kt`'s only literal is `"●"` (kept). Server-provided `device.name` and the dynamic
`extraInfo`/`lastError` suffixes stay literal.
